This article describes the GitHub flow using shared repository.

The pull request is used for review processing.

Tools

http://www.sourcetreeapp.com/

Clone repository

git clone http://192.168.73.211/ivanji/hello-git.git

Configure username and email address

Use global config

git config --global user.name "Jiayi Zhou"
git config --global user.email "jiayizhou@domain.com"

Use repository config

git config user.name "Jiayi Zhou"
git config user.email "jiayizhou@domain.com"

Create new branch for developing new feature

git checkout master
git checkout -b feature1

Edit and commit the changes

git add <edited files>
git commit

GitHub flow

  1. Create a branch from master for developing new feature.

    sh git checkout master git checkout -b awesome-feature

  2. Create, edit, rename, move, or delete files.

    sh git add <edited files> git commit

  3. Push the feature branch to the remote repository.

    sh git push origin awesome-feature

  4. Send a pull request from your branch with your changes to kick off a discussion.

  5. Make changes on your branch as needed. Your pull request will update automatically.

    sh git push origin awesome-feature

  6. Ask partners to merge the pull request, once the branch is ready to go, using the big green button.

  7. Tidy up your branches using the delete button in the pull request or on the branches page.
