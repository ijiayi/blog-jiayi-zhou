Using Github Issues Effectively

Release

Use semantic versioning to make version numbers essentially useful for dependency management.

e.g.,

v1.0.0
v2.3.4
v0.2-alpha
v5.9-beta.3

# Pre-release
v1.0.0-alpha
v1.0.0-alpha.1
v1.0.0-0.3.7
v1.0.0-x.7.z.92

# Build metadata
v1.0.0-alpha+001
v1.0.0+20130313144700
v1.0.0-beta+exp.sha.5114f85
